01 — Problem

Filings are machine-readable in format. They are not machine-readable in interpretation.

An AI agent that asks "does Enron's 2000 10-K show accounting red flags, with citations?" today has three poor options. None of them ship structured, cited judgment.

Status quo failure modes

  • Terminals (Bloomberg, S&P CIQ, FactSet, LSEG) — human-first, API-as-afterthought. They expose fields, not interpretations.
  • AI search (AlphaSense, BamSEC) — finds passages, doesn't structure judgment. "manipulator_likely with 8 cited components" is not a query they answer.
  • Foundation models direct — hallucinate numbers, cannot cite deterministically, no audit trail. Unusable for a CFO, an auditor, or a regulator.

Why now is sharper than two years ago

  • The agent substrate has settled — MCP and OpenAI tool-use specs converged on the same shape.
  • Personal AI agents are real and reaching for finance use cases.
  • OpenAI's GPT-Rosalind launch (life sciences, late 2026) made the shape industry-validated.
  • The category of "domain reasoning + skill orchestration" is now a deployable production pattern, not a research curiosity.
02 — Solution

A skills API for accounting interpretation. Typed. Cited. Deterministic.

Every skill is a callable function with JSON-schema-typed inputs and outputs, per-line-item citations, structured errors, and byte-identical outputs across CLI and API. The agent is the user.

Two stages, deliberately separated

Stage 1. Precomputed, standardized, citation-grounded outputs over a coverage universe. Buyers get the structured interpretation; they do not get the pipeline.

Stage 2. A black-box "skills API" exposing intent-level capabilities. No intermediate artifacts, no chain-of-thought, no pipeline-unit endpoints. Moat-protection by design.

Three operating principles, also product guarantees

P1. Expert layer disjoint from engineering layer. PhDs ship by editing YAML, not by writing Python.

P2. Don't over-engineer, don't be lazy. Zero TODO, zero bare-except, no abstraction without two callers, every function tested.

P3. The user is an AI agent. One manifest renders to MCP + OpenAI tool specs. Errors are typed objects. CLI ↔ API parity is verified.

04 — Skills Catalogue (live, 2026-04-18)

12 skills in the registry today. Same manifests render to MCP + OpenAI tool specs.

Each card lifts the skill's description_for_llm field — the same text an agent reads to choose a skill. Group: fundamental → interpretation → paper-derived → composite.

Fundamental (atomic extraction)

extract_canonical_statementsL1
Three canonical financial statements (IS / BS / CF) for a 10-K, with per-line-item citations to source XBRL or SGML extraction. 16 canonical line items, named consistently across issuers and years.
extract_mdnaL1
Verbatim text of Part II Item 7 (MD&A) from a 10-K. Headings and paragraphs only; HTML markup stripped. Returns null + warning rather than guessing if the section can't be located.

Interpretation (judgment from declarative rules)

interpret_m_score_componentsL2
Per-component natural-language interpretation of an M-score: severity band, accounting-expert-authored text, follow-up questions, citations to the canonical line items that produced each value.
interpret_z_score_componentsL2
Per-component natural-language interpretation of an Altman Z-score: severity band, expert-authored text, follow-up questions, citations (X4's numerator resolves against the market-data fixture).

Paper-derived (faithful implementations of published constructs)

compute_beneish_m_scoreL3
Beneish (1999) earnings-manipulation discriminant; 8 components, 1999 paper-faithful threshold of −1.78 (not the popular −2.22 from Beneish et al. 2013).
compute_altman_z_scoreL3
Altman (1968) bankruptcy-prediction discriminant; 5 components, paper-exact X5 coefficient of 0.999 (not the textbook 1.0).
compute_mdna_upfrontednessL3
Kim, Muhn, Nikolaev & Zhang (2024) MD&A "Upfrontedness" / Information Positioning score. Low score = long, dense paragraphs late in MD&A — a pattern the paper associates with low profitability and high earnings volatility.
compute_context_importance_signalsL3
Kim & Nikolaev (2024) §5.4 firm-year context-importance signals. 4 sub-signals (loss, vol, accruals, MTB extremity); paper-derived weights from Table 7 Panel A.
compute_business_complexity_signalsL3
Bernard, Cade, Connors & de Kok (2025) Section 4 Table 3 Panel a monitoring-demand signals: size, revenue stability, corporate-overhead intensity. Paper-derived weights.
compute_nonanswer_hedging_densityL3
de Kok (2024) OA 3 78-token hedging keyword filter (7 trigrams + 23 bigrams + 45 unigrams) applied to MD&A. Substrate-port warning on every call.
predict_filing_complexity_from_determinantsL3
Bernard, Blankespoor, de Kok & Toynbee (2025) Table 3 Column 2 OLS determinants regression. 5 of 11 paper regressors shipped with paper-exact coefficients (10K=+0.014, Size=+0.012, …).

Composite (orchestrations)

analyze_for_red_flagsL4
Single-call combined Beneish M-score + Altman Z-score screen. Returns both result blocks with per-component interpretations, citations, and full provenance. The headline demo.

05 — Moat & Defensibility

Four pillars. Each testable today. Together they describe a wedge competitors cannot easily replicate.

The wedge is expert judgment + citation provenance + agent-native surface. It is small. That is the right size.

1 · Expert judgment encoded in reviewable YAML

P1 contract. PhDs author rule templates, gold cases, and persona configs as declarative artifacts. No Python required. A real practitioner can replace the accounting_expert subagent persona by editing one YAML file.

2 · Citation/provenance on every claim

Every numeric output carries (doc_id, statement_role, line_item, value, sha256). 213 live citations resolving 100% on the 5-issuer corpus today. A single broken citation blocks release.

3 · Paper-faithful L3 skills

Beneish threshold = −1.78 (the 1999 paper, page 16) — not −2.22. Altman X5 = 0.999 — not 1.0. Each pinned by regression test. The discipline is the difference between an analyst who has read the paper and one who has read a blog post about the paper.

4 · Agent-native by design

One manifest, two catalogs (MCP + OpenAI tool-use), three rendering targets. Natural-agent test passes: a cold Claude given only the MCP catalog solves the Enron 2000 question without hand-holding.

06 — How It Works for Agents

One manifest renders to MCP + OpenAI tool specs. Errors are typed. CLI ↔ API parity is verified.

Every design decision flows from "the user is an AI agent." Three concrete consequences:

1 · Drop-in catalogs

curl -s localhost:8000/mcp/tools    | jq length    # => 12
curl -s localhost:8000/openai/tools | jq length    # => 12

Same 12 skills, two catalogs, derived from one manifest each. No hand-maintained drift.

2 · Structured errors, every time

{
  "error_code": "missing_prior_year_filing",
  "error_category": "MISSING_INPUT",
  "human_message": "The Beneish M-score requires both year t and year t-1 filings; year t-1 (1999-12-31) was not found for CIK 0001024401.",
  "retry_safe": true,
  "suggested_remediation": "Run `mvp ingest filings --cik 0001024401 --years 1999` and retry."
}

No HTTP 500s. No raw exceptions across the skill boundary. Agents recover without human intervention.

3 · The natural-agent test (the headline gate)

A cold Claude given only the MCP catalog and the prompt "analyze whether Enron's 2000 10-K shows accounting red flags, with citations" converged on analyze_for_red_flags with the correct inputs {cik: "0001024401", fiscal_year_end: "2000-12-31"}. All 4 sub-criteria — skill_selection, input_schema, output_usefulness, error_recoverability — PASS. If this gate had failed, the surface would not be agent-native and the MVP would not be done.

08 — Competitive Landscape

The wedge is small. That is on purpose.

The category — machine-readable, audit-grade accounting interpretation as agent-callable skills — has no dominant incumbent. Adjacent categories do.

Gotchas we've already priced in

  • Industry-specific schemas (banks, insurance, REITs need their own canonical mappings).
  • Foreign filers (20-F / 40-F) with different statement structures.
